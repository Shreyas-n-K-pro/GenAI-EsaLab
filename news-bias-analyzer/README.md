# News Bias Analyzer — Data Ingestion Layer

## Overview

This is the **data ingestion layer** for the AI-Powered News Bias and Perspective Analyzer MVP.
It loads, cleans, and unifies three data sources into a single DataFrame that downstream modules
(`bias_engine.py`, `summary_engine.py`, `factcheck_engine.py`, `viz.py`) can consume directly.

## Quick Start

```bash
# 1. Install dependencies
pip install pandas

# 2. Run the data loader (builds cleaned data files)
python data_loader.py

# 3. Run tests to verify everything works
python test_data_loader.py
```

## Data Sources

### 1. AllSides Media Bias Dataset (`allsides_bias_ratings.csv`)
Maps 30 major news outlets to their bias rating: Left, Lean Left, Center, Lean Right, Right.

| Column | Description |
|--------|-------------|
| `name` | Publisher name (e.g., "CNN", "Fox News") |
| `bias_rating` | One of: Left, Lean Left, Center, Lean Right, Right |
| `agree` / `disagree` | Community feedback votes |
| `agree_ratio` | Agreement percentage |

**To use real data:** Download from [AllSides](https://www.allsides.com/media-bias/ratings) or Kaggle and replace the CSV in `data/raw/`.

### 2. HuffPost News Category Dataset (`huffpost_news_sample.json`)
50 news articles across 7 categories in JSON-lines format (one JSON object per line).

| Column | Description |
|--------|-------------|
| `headline` | Article headline |
| `short_description` | Brief summary |
| `category` | POLITICS, TECHNOLOGY, BUSINESS, WELLNESS, ENTERTAINMENT, ENVIRONMENT, TRAVEL |
| `authors` | Author name(s) |
| `link` | Original URL |
| `date` | Publication date |

**To use real data:** Download from [Kaggle](https://www.kaggle.com/datasets/rmisra/news-category-dataset) and replace the JSON file.

### 3. Multi-Publisher Article Corpus (`multi_publisher_articles.json`)
27 articles across 5 topics, each covered by 5-6 different publishers with varying bias. This is the **primary dataset for perspective comparison**.

**Topics included:**
- `budget_2024` — Federal Budget Battle (6 publishers)
- `immigration_reform` — Immigration Policy Reform (6 publishers)
- `climate_policy` — Climate Change Policy (5 publishers)
- `ai_jobs` — AI Impact on Employment (5 publishers)
- `healthcare_costs` — Healthcare Affordability (5 publishers)

## Unified Output Schema

All data is merged into a single DataFrame with this schema:

| Column | Type | Description |
|--------|------|-------------|
| `title` | str | Headline / article title |
| `publisher` | str | News outlet name |
| `article_text` | str | Body or short description |
| `category` | str | Topic category (uppercase) |
| `bias_label` | str | AllSides-style bias label |
| `url` | str | Original article URL |
| `published_date` | str | Date in YYYY-MM-DD format |
| `topic_id` | str/NaN | Shared topic ID (for multi-publisher stories) |

## API Reference

### Core Loading Functions

```python
from data_loader import load_all_data, get_bias_map

# Load the unified dataset (all sources merged)
df = load_all_data()

# Get the publisher -> bias label mapping
bias_map = get_bias_map()
# Returns: {"CNN": "Lean Left", "Fox News": "Right", ...}
```

### Query Helpers (for other modules)

```python
from data_loader import (
    get_articles_by_topic,
    get_available_topics,
    get_categories,
    get_bias_distribution,
)

# Get articles about a specific topic, sorted Left -> Right
topic_df = get_articles_by_topic("budget_2024")

# List all available topics (useful for UI dropdowns)
topics = get_available_topics()

# Get all category names
categories = get_categories()

# Get bias distribution across all articles (for charts)
dist = get_bias_distribution()
```

### Export

```python
from data_loader import export_cleaned_data

# Save cleaned files to data/cleaned/
paths = export_cleaned_data()
# Returns: {"unified_articles": "path/to/csv", "bias_map": "path/to/json", ...}
```

## For Teammates

### Person 2 (Bias Engine)
Import `get_bias_map()` and `get_articles_by_topic()`. The bias map gives you publisher -> label lookup. Topic articles come pre-sorted Left to Right.

### Person 3 (Summary Engine)
Import `get_articles_by_topic()` to get all perspectives on a topic. The `article_text` column contains the full text for summarization.

### Person 4 (Visualization)
Import `get_bias_distribution()` for charts and `get_available_topics()` for the UI sidebar.

## Assumptions

1. Sample data is included for demo purposes. Replace with real datasets for production.
2. HuffPost articles are all labeled as "Left" (their AllSides rating).
3. The multi-publisher corpus contains realistic but synthetic article text.
4. Publisher name matching is case-sensitive but whitespace-tolerant.
5. Categories are normalized to uppercase.
